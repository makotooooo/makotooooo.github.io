ES6-Symbol

1.概述

Symbol是 ES6 中新的数据类型，表示独一无二值。它是 JavaScript 语言中的第七种数据类型，前六种是： undefined 、null、布尔值（Boolean）、字符串（String）、数值（Number）、对象（Object）。

Symbol值通过Symbol函数生成。

注意，Symbol函数不能使用new命令，否则会报错，这是因为生成的Symbol是一个原始数据类型，不是一个对象。也就是说，由于Symbol值不是对象，所以不能添加属性。基本上，它是一种类似于字符串的数据类型。

由于Symbol是一种基础的数据类型，所以当我们使用typeof去检查它的类型的时候，它会返回一个属于自己的类型Symbol，而不是string，object之类的

另外你也可以在调用Symbol()的时候传一个参数，但这个参数的表示对Symbol实例的描述，主要是为了在控制台显示或者转为字符串时，增加可读性。

我们需要重点记住的一点是：每个Symbol实例都是唯一的，当比较两个 Symbol 实力的时候，将总会返回 false。Symbol 函数的参数只是对当前 Symbol 实例的描述，因此相同参数的 symbol 函数的返回值是不想等的

不幸的是，与该对象交互的代码依然可以访问其键为 symbol 的属性。甚至在不能直接访问到 Symbol 本身的情况下拿到其值，例如 Object.getOwnPropertySymbols()方法

以及新增的反射 API

注意:目前正在做一些工作来处理在 JavaScript 中向类添加私有属性的问题。这个特性的名称被称为私有字段，虽然这不会使所有对象受益，但会使类实例的对象受益。私有字段从 Chrome 74 开始可用。

使用 Symbol 为 JS 对象提供私有属性可能有点困难，但 Symbol 还有另一个好处，就是避免不同的库向同一个对象添加属性名产生冲突的问题。

考虑这样一个场景，两个不同的库想要向同一个对象添加一些基本数据，可能他们都想在这个对象上设置某标识符。例如使用 id 作为键，这样就存在一个巨大的风险，就是多个库使用相同的键。

通过使用 Symbol，每个库可以在实例化时生成所需的 Symbol。然后用生成 Symbol 的值作为对象的属性。

这样就可以保证每个库添加的属性都是唯一的，不可能重复。而且不会被遍历或者序列化输出，保证了一定的隐秘性。

但是你可能会问，在没有 Symbol 时这些也是能实现的。

我们可以使用随机字符串或者命名空间作为对象的键，这样就保证了两个库不使用相同的命名空间键名就不会重复，然后再设置属性时设置 enumerable 为 false，这样就可以防止该属性被遍历出来，在这一点上我们几乎重新实现了 Symbol 的功能，而且这两种方式都可以使用 Reflect.ownKeys()方法读取，因此他们并不是私有的。

但由于字符串时不可变的，而 Symbol 总是保证唯一的，所以仍然可能会有极低的可能生成相同的字符串，从数学概率上讲，Symbol 的确提供了字符串无法比拟的好处。
