环境变量
发表于 2020-08-03
阅读量:

随便翻开一本 java 教程，第一步一般都是要教你配置环境变量，那为什么要配置环境变量？环境变量为什么叫这个名字呢？

一般配错了也只是知道 javac 用不了了，但为什么会用不了呢？我们可能不会理解其中的原因，如果搞清楚原理的话，我们就能更从容的应对这样的问题

环境变量

环境变量（environment variables）一般是指在操作系统中用来指定操作系统运行环境的一些参数，如：临时文件夹位置和系统文件夹位置等。

我们设想一个场景来进行说明，你需要运行一个软件，第一件事是什么？

是需要先找到这个软件在哪，然后去点击他运行。对于计算机来说也是一样的，我们在命令行里写的指令电脑的第一件事也是去寻找他在哪里，然后执行。

那计算机会去哪些地方寻找软件或指令呢？

首先计算机会在当前目录下寻找，如果没有找到就会去 Path 中存储的路径寻找，如果找到就执行，没有找到就会提示命令不存在

每次输入指令之前，把当前指令的文件路径页输入进去也可以使用，但这很明显太麻烦了

这就要介绍我们的 Path 变量，我们在命令行中写完指令之后计算机会去遍历 Path 中的地址，如果找到就执行，没有找到就会提示命令不存在。

Java 环境变量

初次安装 java 之后，为了更方便的使用我们一般都会配置以下三个环境变量。

  • JAVA_HOME：指向 jdk 的安装目录，作用是一些基于 Java 的开发工具可能会用上，比如 Tomcat，Eclipse 等，如果不配置JAVA_HOME的话这些软件会找不到 jdk，换句话说如果不用这些软件JAVA_HOME不配也是没有关系的。
  • Path：正如上文所说，计算机会去 Path 中寻找指令来运行，所以这里就需要将其指向 jdk 下的bin目录，bin目录中有编译，启动（javac，java）等，所以为了这些命令可以在任何地方使用，而不用输入长长的路径，我们需要在 Path 中添加 %JAVA_HOME%/bin
  • CLASSPATH：在于告诉 Java 执行环境，在那些目录下可以找到我们所要执行的 Java 程序所需要的类或者包。不过在 JDK1.5 之后不配置该环境变量也能正常运行，但网上很多 1.7，1.8 版本的教程还在说需要配置，所以只要多了解一些原理，就能知道这样做的用处

Maven 环境变量

初次安装 Maven 之后也需要我们配置环境变量

网上的教程比较混乱，有说配MAVEN_HOME的也有说配M2_HOME的，配置完该变量，再将其引入到Path中，据说MAVEN_HOME是 maven1 的时候配置，M2_HOME是 maven2 以上的版本配置（maven3 也是M2_HOME，注意不是M3_HOME）

不过官方教程对于M2_HOME只字未提

官方的意思是就是不用配置任何的环境变量，只需要将其路径直接加入Path即可。
